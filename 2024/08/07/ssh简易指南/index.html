ssh概览指南

Posted on 2024-08-07 Edited on 2025-07-10 In 网络 Views:
Symbols count in article: 4.2k Reading time ≈ 15 mins.

Secure Shell (SSH) 是一个允许两台电脑之间通过安全的连接进行数据交换的 网络协议。通过加密保证了数据的保密性和完整性。SSH采用 公钥 加密技术来验证远程主机，以及 (必要时) 允许远程主机验证用户。

本文将介绍一些ssh简单的使用方式，包括登录方式，密钥设置，白名单设置等简单的配置。

ssh登录

SSH 登录方式总览

SSH（Secure Shell）支持多种登录方式，不同方式适用于不同的使用场景。下面是 SSH 登录的主要方式：

登录方式 是否推荐 说明
1️⃣ 密码登录 ❌ 不推荐 使用用户名和密码直接登录，易受暴力破解攻击
2️⃣ 公钥认证（RSA/ECDSA/Ed25519） ✅ 推荐 使用本地密钥对，安全且自动化
3️⃣ SSH Agent 转发 ✅ 推荐（高级场景） 使用本地 SSH Agent 来远程使用密钥
4️⃣ 单点登录（SSO） ✅ 推荐（企业级） 使用 Kerberos、LDAP 等做统一认证
5️⃣ 多因素认证（MFA） ✅ 推荐 密码 + 动态口令 / YubiKey / 手机认证
6️⃣ 证书登录（CA 签发 SSH 证书） ✅ 推荐（企业高级） 使用 SSH CA 签发的证书认证
7️⃣ 基于跳板机（Jump Host） ✅ 推荐 通过中间跳板服务器登录目标主机
8️⃣ 动态口令登录（OTP） ⚠️ 可选 一次性密码验证器（如 Google Authenticator）
9️⃣ 登录 shell 服务器（如 GitHub CLI） ⚠️ 可选 用于 GitHub 等平台的远程 CLI 登录

公钥认证

在本地通过加密算法生成一个密钥对（公钥+私钥），将公钥上传自服务器。然后通过私钥登录，ssh server通过保持的公钥来验证对应的私钥。之后就可以免密登录目标主机了。

生成密钥对

ssh-keygen命令用于生成密钥，通过-t参数可以指定密钥类型，rsa 表示使用 RSA 算法（还有 ed25519, ecdsa, dsa 等）。-b 4096指定密钥长度（单位bit，需要配和rsa使用），-f指定生成的公钥和私钥文件名称（私钥文件不带后缀名，公钥文件带.pub后缀名）。

常见密钥算法对比及使用示例

算法 安全性 🔒 速度 ⚡ 密钥体积 📦 备注
RSA 高（≥2048位） 慢（密钥越长越慢） 大（4096位较大） 兼容性最好
ECDSA 高（但依赖库实现质量） 快 小 在某些平台兼容性不佳
ED25519 非常高 非常快 非常小 安全、轻便、现代推荐
DSA 已弃用 还行 小 ❌ 不推荐

1️⃣ ED25519（现代首选）

  • 使用 Ed25519 算法生成密钥
  • 快速、小巧、默认抗侧信道攻击
  • 推荐给 GitHub、Linux、日常 SSH 登录等

2️⃣ RSA（2048位）（兼容首选）

或更安全一点：

  • 适用于所有平台（Windows、Linux、嵌入式等）
  • -b 指定位数（4096 安全性更高，但速度慢些）

3️⃣ ECDSA（椭圆曲线）

  • 位数可选：256、384、521（常用 256）
  • 比 RSA 快，占用更少空间
  • 依赖 OpenSSL 实现质量，部分设备不支持

4️⃣ DSA（不推荐）

  • 密钥固定为 1024 位，已被 OpenSSH 弃用
  • 不支持 authorized_keys 选项和新加密标准

参考SSH 证书登录教程

在上面的登录过程中，使用密码登录有密码泄露的安全风险。而通过公钥登录，公钥默认是永久有效的，除非在authorized_key文件中将对应的公钥删除。而使用SSH CA证书来登录可以为用户公钥设置有效期来更好管理用户登录。

使用 SSH CA 签发用户证书的场景

  • 更灵活的密钥生命周期管理（证书可以设置有效期）

  • 撤销用户只需要更新信任列表（CA 公钥）

  • 支持更复杂的权限控制（如特定命令、主机、身份）

  • 多个用户公钥都可以由同一个 CA 签发

ssh管理

密钥管理

ssh-agent

ssh-agent 是一个用于管理私钥并在后台自动使用它们进行身份验证的程序。它让你只输入一次私钥的解锁密码，之后自动帮你在 SSH 登录中使用这些密钥，不用每次都手动输入密码或指定私钥路径。

ssh-agent 的作用
功能 说明
🔐 缓存私钥 缓存你添加的私钥（包括带密码的）
🧠 记住解锁密码 你只需输入一次私钥密码，之后自动使用
🧳 跨多次 SSH 登录 多个终端、跳板机都可复用密钥
🧩 与 ForwardAgent 结合 可以在跳板机中使用本地密钥
🔒 不需要把密钥拷贝到跳板机上 提高安全性，防止泄露私钥
