nginx概览指南

Posted on 2025-07-07 Edited on 2025-07-14 In 工具 Views:
Symbols count in article: 2.5k Reading time ≈ 9 mins.

Nginx（发音为“Engine-X”）是一个高性能、轻量级的 Web 服务器和反向代理服务器，也常被用于负载均衡、HTTP 缓存等场景。

Nginx 的主要作用

作用类别 说明
Web服务器 提供静态网页服务（如 HTML、CSS、JS、图片等），相比 Apache 占用资源更少，处理并发能力更强。
反向代理服务器 接收客户端请求，将其转发给后端服务器（如 Node.js、Python、PHP 服务），客户端只看到 Nginx，不直接访问后端。
负载均衡器 分发请求到多个后端服务，实现高可用、高并发网站架构。支持轮询、IP hash 等多种分发算法。
HTTPS终端服务器 管理 TLS/SSL 证书，实现 HTTPS 通信。Nginx 可以处理 HTTPS，后端只用处理 HTTP。
缓存服务器 可缓存后端响应结果（如页面、API响应），提升性能、减轻后端压力。
反向代理 + Web应用防火墙（WAF） 与防火墙模块配合使用可拦截恶意请求、实现访问控制等。

配置文件结构

Nginx配置文件一般由全局块和代码块组成。

http，stream等这些块被称为顶层块，用于不同的功能：

顶层块名 功能描述 是否默认支持 常见用途
http HTTP/HTTPS 请求处理 ✅ 默认启用 提供网页、反向代理、负载均衡、静态资源服务等
stream TCP/UDP 流量代理 ❌ 需编译支持 --with-stream TCP 代理、MySQL、Redis、WebSocket（低层）
mail 邮件协议代理 ❌ 需编译支持 --with-mail SMTP/POP3/IMAP 代理（如做邮件网关）
events 连接处理模型设置 ✅ 默认启用 设置连接数、事件驱动模型（如 epoll）

另外还有server块、location等块，他们和顶层块一起统称为上下文块（父级块），如下所示：

父级块名 所在层级（可嵌套位置） 用途说明
main（无名字） 顶层，全局作用 Nginx 启动配置，如 user、worker_processes 等
events 顶层（global） 设置连接处理模型与并发连接数
http 顶层（global） 提供 HTTP 协议服务，如反向代理、缓存、虚拟主机等
server http 内部 定义虚拟主机，监听端口、配置域名
location server 内部 匹配 URL 路径并定义其处理方式
upstream http 内部 配置后端服务器组，实现负载均衡
mail 顶层（global） 提供邮件代理服务（SMTP/POP3/IMAP）
server（邮件） mail 内部 定义邮件协议、监听端口、验证机制
stream 顶层（global） 提供 TCP/UDP 代理服务（如数据库、WebSocket）
server（stream） stream 内部 TCP/UDP 虚拟主机配置

Nginx反向代理

反向代理是Nginx最主要的功能，通过Nginx反向代理可以接受客户端请求，并将其转发给后端服务器，这样客户端不会直接接触后端服务，由 Nginx 统一处理流量、缓存、负载均衡、安全控制等。

反向代理常见场景

场景 说明
前后端分离 前端请求转发给后端 API 服务
多服务聚合 将不同路径转发到不同后端服务
负载均衡 多个后端服务分担压力
统一入口网关 提供单一域名，对接多个后端系统
屏蔽后端 IP 隐藏真实后端服务器地址，提升安全性

什么是正向代理，什么是反向代理

  • 正向代理：客户端知道目标地址，但通过代理服务器中转请求（这样服务器就不知道真实的访问服务器的客户端地址了）

    比如说翻墙这种，客户知道目标服务器地址但是被限制访问，通过代理服务器转发请求可以伪装身份或突破限制。

  • 反向代理：客户端不知道真实后端地址，只和代理服务器交互

    比如Nginx 作为反向代理根据请求路由决定转发到哪个后端，用户永远只看到 www.example.com。

反向代理相关的指令都在location块中，通过proxy_*等指令来设置代理相关的功能。

反向代理指令

✅ 核心反向代理指令

指令 功能说明 示例
proxy_pass 将请求转发给后端服务器 proxy_pass http://127.0.0.1:3000/;
proxy_set_header 设置请求头，传递客户端信息 给后端 proxy_set_header Host $host;
proxy_redirect 设置返回响应头中的 Location 重写 proxy_redirect off; 或 proxy_redirect http://127.0.0.1:3000/ /;

⚙️ 常见辅助指令（性能优化、控制行为）

指令 功能
proxy_http_version 设置代理请求使用的 HTTP 版本（如 1.1，启用 WebSocket 支持时常用）
proxy_connect_timeout 设置连接后端服务器的超时时间
proxy_read_timeout 设置读取后端响应的超时时间
proxy_send_timeout 设置向后端发送请求的超时时间
proxy_buffering 是否启用响应缓存，默认 on
proxy_cache 使用缓存区域，提高响应速度
proxy_cache_valid 缓存某状态码的响应时间
proxy_next_upstream 后端出错时是否自动切换下一个服务器
proxy_intercept_errors 是否拦截后端错误，使用自定义错误页面

✅ 总结

类别 指令示例
核心 proxy_pass, proxy_set_header, proxy_redirect
超时控制 proxy_connect_timeout, proxy_read_timeout
缓存优化 proxy_buffering, proxy_cache, proxy_cache_valid
错误处理 proxy_intercept_errors, proxy_next_upstream
协议支持 proxy_http_version, proxy_set_header
